# 项目开发计划

## 项目定位

本项目是一个 `AI 求职闭环工作台`，目标不是提供单次简历打分，而是围绕用户完整求职流程，持续提升以下核心指标：

- 被查看率
- 邀约率
- 入职率

产品将求职流程拆成 4 个阶段：

1. 准备
2. 匹配
3. 投递
4. 面试与复盘

系统当前采用 `Nuxt 3 + Vue 3 + TypeScript + Naive UI`，后端分析链路优先接入 `DeepSeek API`，失败时回退本地 mock 分析逻辑。

说明：
- 将准备阶段中的目标岗位、项目证据、自我介绍底稿同步到 `/workspace`。

## 未来展望

状态：`中长期规划`

### 1. 从“工具”升级为“求职操作系统”

- 不只是做一次分析，而是保存用户的长期求职轨迹。
- 逐步形成材料、岗位、投递、面试、复盘的一体化数据链路。

### 2. 增强 AI 生成与编辑能力

- 支持多版本简历自动改写。
- 支持岗位定制版自我介绍。
- 支持项目经历从长文本自动整理为 STAR / CAR 格式。

### 3. 建立可量化的转化追踪

- 用户可以知道：
  - 哪版简历查看率更高
  - 哪类岗位邀约率更高
  - 哪种自我介绍结构更有效

### 4. 建立阶段化入口体系

- `/journey/prepare`
- `/journey/match`
- `/journey/apply`
- `/journey/interview`

让每个阶段真正成为独立模块，而不是说明页。
