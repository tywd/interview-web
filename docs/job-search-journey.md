# 求职步骤说明

本产品将求职流程拆成 4 个阶段，目标不是单次打分，而是帮助用户持续提升被查看率、邀约率和入职率。

## 1. 准备

### 大目标

- 明确目标岗位、城市、职级和行业范围。
- 准备一份可投递的基础简历和项目清单。
- 收集目标岗位 JD 与公司资料，形成分析输入。

### 执行小点

- 把项目经历改成“目标、动作、结果”结构。
- 优先补充量化指标、协作角色和业务影响。
- 建立一份能持续迭代的基础求职档案。

## 2. 匹配

### 大目标

- 识别岗位关键词、核心职责和加分项。
- 判断候选人的技术、业务和协作经验匹配度。
- 找到影响被查看率和邀约率的表达短板。

### 执行小点

- 区分“能力不足”和“表达不足”这两类问题。
- 把 JD 语言映射到项目经历和技能标签里。
- 优先修复高频风险：不量化、不贴岗、不懂业务。

## 3. 投递

### 大目标

- 基于分析结果生成岗位定制版简历。
- 围绕岗位、公司和业务场景做投递版本管理。
- 记录每次投递状态，持续优化命中率。

### 执行小点

- 为不同公司保留多份简历与版本快照。
- 优先投高匹配岗位，降低无效投递消耗。
- 把 AI 结论转化为更具体的简历修改动作。

## 4. 面试与复盘

### 大目标

- 准备自我介绍、项目讲解和常见追问。
- 围绕岗位要求生成更贴题的回答框架。
- 每轮面试后复盘并沉淀下轮优化动作。

### 执行小点

- 针对技术面、业务面、HR 面分别准备素材。
- 记录被追问点、卡壳点和 offer 风险项。
- 持续提升最终入职率，而不是只追求面试次数。
